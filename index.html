Plunder Panic

Plunder Panic is a pirate-themed, up to 12-player, action-arcade game where two teams race to one of four win conditions. In 2017 it attended IndieCade and won the Audience Choice Award. Development continued with a substantial graphics overhaul and the addition of AI bots. In 2019 the game was part of the IndieCade Esports showcase, and it was released on Steam in September 2021.

View on Steam ↗

I began work on Plunder Panic in May of 2019, brought on to develop AI bots and rework existing gameplay features and systems.

Contributions

  • Designed and created a dual-utility AI for decision making among bots
  • Created a node-based movement system and used a modified A* algorithm for pathing
  • Designed and created an interchangeable action system for developing bot behavior
  • Implemented a universal data tracking system to help track and balance victories
  • Utilized a virtual controller to create bot behavior and allow for easier networking implementation
  • Designed and created new character select scene with variations for both Arcade and Nintendo Switch versions
  • Implemented new menu system featuring updated art
  • Implemented new game startup loop and pause functionality
  • Reworked functionality of the heavy gold chest item to prevent imbalanced gameplay

Whipslash

In the far future, you find yourself trapped in a complex full of extremely hostile robots. Armed with two electric axes, you must fight your way out. Whipslash was a project for a Game Design Studio course at Michigan State University and was accepted as one of the 550 games competing in the 2020 Independent Games Festival.

I served as producer and a gameplay/systems programmer with a focus on enemy interaction.

Contributions

  • Implemented a Kung-fu circle attack system to keep enemies in front of the player without overwhelming them
  • Designed and developed a grapple movement system to traverse the large environment
  • Added aim-assisted targeting for axe throwing mechanic
  • Created a ragdoll system using character joints to allow the player to tear apart grunt enemies
  • Implemented and enhanced a generic audio manager system
  • Managed a team of 10 students throughout a semester, keeping the game within scope
  • Rescoped project when necessary to ensure the team hit desired goals without being overworked
  • Created a character-driven responsive audio system to dynamically transition between faster and slower sections of the same song
